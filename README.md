# auto-speakers

`spkreg` — маленькая утилита, которая **узнаёт голоса** в записях ваших рабочих встреч.

## Что это и зачем

Программы для расшифровки встреч умеют делить запись на реплики и отмечать, что говорили разные люди — но называют их безлично: `SPEAKER_00`, `SPEAKER_01`. Причём на каждой новой записи нумерация своя: сегодняшний `SPEAKER_00` на завтрашней встрече легко станет `SPEAKER_02`.

`spkreg` это чинит. Вы один раз указываете, какой голос кому принадлежит — и дальше на любых новых записях программа сама подсказывает: «этот голос на 96% похож на Ивана». Имена перестают путаться, а расшифровку можно сразу читать с настоящими именами вместо `SPEAKER_00`.

## Как это работает

Диаризатор (программа, которая делит запись по голосам) не только режет разговор на реплики, но и считает для каждого голоса **отпечаток** — набор из 256 чисел, описывающий тембр и манеру речи. У одного человека отпечатки на разных записях выходят похожими, у разных людей — заметно расходятся.

`spkreg` делает с этими отпечатками три вещи — **сохраняет и накапливает**, **усредняет** и **сравнивает**:

1. **Сохраняет и накапливает.** При обучении вы даёте программе отпечатки и явно говорите, кто есть кто. Разметить голоса можно двумя способами: либо через языковую модель (LLM), попросив её определить спикеров по репликам, либо вручную — самому проставив соответствие. Результат записывается в текстовый файл, где каждому имени соответствует несколько отпечатков, накопленных из разных расшифровок.

2. **Усредняет.** По всем образцам одного человека программа считает средний отпечаток — эталон. Такой эталон лучше любого отдельного образца и не зависит от случайностей одной записи: шума микрофона, простуды, плохой связи. А если в расчёт попал ошибочный отпечаток — его всегда можно убрать из базы по этому человеку: исходные данные накапливаются и в любой момент редактируются.

3. **Сравнивает и ищет соответствие.** Когда эталонов накопилось достаточно, включается автоматическое узнавание. Каждый отпечаток удобно представлять как стрелку-направление в пространстве из 256 измерений: у похожих голосов стрелки смотрят почти в одну сторону, у разных — расходятся. Программа меряет не длину стрелок, а угол между ними — то есть само направление голоса, а не его громкость или длину записи. Угол сворачивается в одно число — близость: около 0.99 у одного и того же человека, около 0.44 у разных. Для каждого голоса из новой записи программа находит ближайшие по углу эталоны и предлагает кандидатов на выбор. Дальше за вами — выбрать нужного и переразметить текстовый файл.

## Установка

Нужны две вещи: диаризатор, который делает отпечатки, и сам `spkreg`.

### 1. Диаризатор

`spkreg` не работает со звуком напрямую — ему нужна готовая расшифровка. Подойдёт любой диаризатор, лишь бы он отдавал JSON вот такого вида: на каждую реплику — время, метка говорящего и текст, а рядом — голосовой отпечаток на каждого говорящего.

```json
{
  "segments": [
    {"start": 0.0, "end": 2.3, "speaker": "SPEAKER_00", "text": "Привет, Мария."},
    {"start": 2.3, "end": 5.1, "speaker": "SPEAKER_01", "text": "Здравствуй, Иван."}
  ],
  "speaker_embeddings": {
    "SPEAKER_00": [0.11, -0.07, 0.24, ...],
    "SPEAKER_01": [0.02,  0.19, -0.05, ...]
  }
}
```

Главное здесь — блок `speaker_embeddings`: по одному вектору на говорящего (это и есть отпечатки; в примере они сокращены, на деле — по 256 чисел). Без него узнавание не заработает.

Подходящий диаризатор с такой выдачей — **whispermlx** (<https://github.com/KalebJS/whispermlx>). Установка (Python 3.12):

```bash
uv tool install --python 3.12 whispermlx \
  --with "numba>=0.61" --with "torchcodec>=0.6.0,<0.8.0"
```

Модель, которой whispermlx различает голоса, закрыта согласием — её нужно один раз открыть:

1. Заведите бесплатный аккаунт на <https://huggingface.co>
2. Откройте страницу модели <https://huggingface.co/pyannote/speaker-diarization-community-1> и примите условия использования (кнопка на странице).
3. Создайте токен доступа на <https://huggingface.co/settings/tokens> и добавьте его в окружение:
   ```bash
   export HF_TOKEN=hf_ваш_токен
   ```

После первого запуска модель скачается на компьютер, и токен больше не понадобится.

### 2. spkreg

Любой способ на выбор.

**Просто скопировать скрипт** — нужен только Python 3, никаких зависимостей:

```bash
curl -fsSL https://raw.githubusercontent.com/mainpart/auto-speakers/main/spkreg.py -o ~/.local/bin/spkreg
chmod +x ~/.local/bin/spkreg
```
(каталог `~/.local/bin` должен быть в `PATH`)

**Через pip или uv:**

```bash
pip install git+https://github.com/mainpart/auto-speakers
# или в отдельном изолированном окружении:
uv tool install git+https://github.com/mainpart/auto-speakers
```

**Через Homebrew:**

```bash
brew tap mainpart/auto-speakers https://github.com/mainpart/auto-speakers
brew trust mainpart/auto-speakers
brew install spkreg
```

## Как пользоваться

**Шаг 1. Сделать расшифровку с отпечатками.** Прогоните запись через диаризатор — обязательно в формате JSON (`-f json`): отпечатки голосов whispermlx кладёт только в JSON, форматы txt и srt для `spkreg` не годятся.

```bash
whispermlx встреча.mp3 --language ru --diarize --speaker_embeddings -f json --output_dir .
```

Получится файл `встреча.json` с репликами и отпечатками голосов. Голоса в нём пока безымянные: `SPEAKER_00`, `SPEAKER_01`…

**Шаг 2. Один раз — представить голоса.** Определите, кто есть кто — по репликам с помощью языковой модели (LLM) или на слух — и передайте программе отпечатки на хранение:

```bash
spkreg enroll встреча.json SPEAKER_00="Иван Иванов" SPEAKER_01="Мария Петрова"
```

Имена с пробелом (имя и фамилия) берите в кавычки, как выше. Пар `SPEAKER_XX=Имя` можно передать сколько угодно за один раз.

**Шаг 3. На новых записях — узнать голоса.** Для каждого голоса программа покажет самого похожего человека, уверенность в процентах и пару запасных вариантов:

```bash
spkreg identify другая-встреча.json
```

**Подставить имена прямо в текст расшифровки:**

```bash
spkreg apply другая-встреча.json -o готово.txt
```

### Все команды

| Команда | Что делает |
|---|---|
| `spkreg enroll файл.json SPEAKER_00="Имя Фамилия" …` | Запоминает, какой голос кому принадлежит. Пар `SPEAKER_XX=Имя` можно передать сколько угодно за раз; имена с пробелом — в кавычках. |
| `spkreg identify файл.json` | Показывает, кто говорил на новой записи: для каждого голоса — самый похожий человек и ещё несколько кандидатов. Ключ `--top N` меняет число кандидатов (по умолчанию 3). |
| `spkreg apply файл.json -o готово.txt` | Сохраняет расшифровку с настоящими именами вместо `SPEAKER_XX`. |
| `spkreg list` | Показывает всех, кого программа уже знает. Ключ `-v` добавляет, из каких файлов набран каждый голос. |
| `spkreg forget Имя` | Забывает человека целиком. |
| `spkreg forget Имя --source файл.json` | У этого человека убирает только образец, пришедший из указанного файла. |
| `spkreg forget --source файл.json` | Убирает образцы из указанного файла — у всех, кто из него набран. |

У `identify` есть ещё ключ `--threshold` — порог близости, ниже которого голос считается незнакомым (по умолчанию `0.70`).

Полезно знать: чем с большего числа **разных** встреч вы представите одного человека, тем точнее программа будет его узнавать. Двух-трёх записей обычно достаточно.

Список голосов хранится в одном файле `~/.local/share/spkreg/registry.json`. Чтобы держать его в другом месте — например, отдельные базы под разные проекты — задайте переменную окружения `SPKREG_HOME`, и `registry.json` появится в указанной папке:

```bash
export SPKREG_HOME=~/Documents/голоса
```
